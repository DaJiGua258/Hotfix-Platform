## 一、编辑器工具（上传）

### 1.1 打开工具

Unity 菜单栏 → `Tool/ABTool`，弹出配置窗口。

### 1.2 配置 FTP

| 字段 | 说明 |
|------|------|
| FTP 地址 | 默认 `ftp://127.0.0.1` |
| 用户名 | 默认 `user` |
| 密码 | 默认 `123456` |
| 本地 AB 包路径 | AB 包所在目录绝对路径 |

配置存储在 `EditorPrefs` 中，不同项目隔离。

### 1.3 操作流程

**① Lua 转 TXT**

- 自动扫描 `Assets/Lua/` 下所有 `.lua` 文件（递归）
- 复制到 `Assets/LuaTxt/`，重命名为 `文件名.lua.txt`
- 设置 AB 标签为 `lua`（用于打包到 lua AB 包）

**② 生成本地 AB 包的对比文件**

- 扫描指定目录下所有无扩展名的文件（即 AB 包文件）
- 为每个文件生成 `包名_文件大小_MD5` 格式的条目
- 写入 `ABCompareFile.txt`

**③ 保存资源到 StreamingAssets**

- 在 Project 窗口中选中要发布的 AB 包文件
- 点击按钮复制到 `Assets/StreamingAssets/`
- 自动重新生成对比文件

**④ 上传到 FTP**

- 遍历 `StreamingAssets` 目录
- 对所有 AB 包文件和 `.txt` 文件执行 FTP 上传
- 异步上传，每个文件分 2KB buffer 分块写入

> **注意**：AB 包的构建不在本工具中，需通过 Unity AssetBundle Browser 或 `BuildPipeline.BuildAssetBundles` API 事先完成。

### 1.4 上传工具代码结构

```
ABUploadTool.cs         → EditorWindow GUI + 上传/转换功能
ABNetInfo.cs            → FTP 配置信息 + MD5 工具方法
```

## 二、运行时（下载）

### 2.1 核心流程

```
GameManager.Start()
    │
    ▼
ABDownload.CheckUpdate()
    │
    ├── 1. DownRemoteCompareFile()
    │     从 FTP 下载 ABCompareFile.txt → 保存为 ABCompareFile_TMP.txt
    │     (使用 TMP 暂存，避免下载中途崩溃破坏本地版本信息)
    │
    ├── 2. 解析远端对比文件
    │     读取 ABCompareFile_TMP.txt
    │     逐行解析 → Dictionary<string, ABInfo>
    │     ABInfo: { Size, MD5 }
    │
    ├── 3. GetCheckLocalCompareFile()
    │     查找本地对比文件
    │     - 优先 persistentDataPath (可读写)
    │     - 回退 streamingAssetsPath (只读)
    │
    ├── 4. CompareLocalAndRemote()
    │     逐包 MD5 对比，生成待下载列表
    │     删除本地有而远端无的冗余包
    │
    ├── 5. DownloadABFiles()
    │     依次 FTP 下载差异 AB 包到 persistentDataPath
    │
    └── 6. 写入本地对比文件
         remoteInfo (即 TMP 内容) 写入 persistentDataPath/ABCompareFile.txt
         取代旧版本信息
```

### 2.2 TMP 对比文件机制

`ABCompareFile_TMP.txt` 是一个**暂存文件**，关键作用在于**崩溃安全**：

```
第一次启动 (无本地版本)：
  ┌─ persistentDataPath/ABCompareFile.txt        → 不存在
  ├─ persistentDataPath/ABCompareFile_TMP.txt     → 不存在
  └─ streamingAssetsPath/ABCompareFile.txt        → 内置的初始版本

更新流程：
  ① 从 FTP 下载 ABCompareFile.txt
              ↓
  ② 保存到 ABCompareFile_TMP.txt  ← 先写 TMP，不覆盖正式文件
              ↓
  ③ 解析 TMP → 对比 → 下载差异包
              ↓   └── 若中途崩溃，正式文件未被破坏，
              │      下次启动仍可使用旧版本作为对比基线
              ↓
  ④ 所有下载成功 → TMP 内容写入 ABCompareFile.txt (正式文件)
     (TMP 文件不会删除，下次更新直接覆盖)

崩溃安全示例：
  - 第 3 次启动 → TMP 已下载但下载差异包时崩溃
  - 第 4 次启动 → 再次下载 TMP → 对比正式文件 → 继续下载未完成的包
  - 正式文件的版本信息始终是"上一次完整更新后的状态"
```

### 2.3 资源加载路径优先级

| 位置 | 说明 | 读写 |
|------|------|------|
| `persistentDataPath` | 下载更新的 AB 包 | 可读写 |
| `streamingAssetsPath` | 打包内置的 AB 包（降级回退） | 只读 |

`ABMgr.GetABPath()` 优先检测 `persistentDataPath` 是否存在，不存在则走 `streamingAssetsPath`。

StreamingAssets vs PersistentDataPath
1. StreamingAssets(只读,打包进安装包)
    - 打包时放入的原始资源目录,随安装包一起分发到设备。只读(Android上实际压缩在APK内,不能用普通文件IO访问,需用 UnityWebRequest 读取)。
    - 用途:内置首包资源,如首包AssetBundle、配置文件、初始清单(manifest)——不用下载,装了游戏就有。
2. PersistentDataPath(可读可写,运行时生成)
    - 设备上应用私有的可写目录,不同平台路径不同,但保证读写权限。
    - 用途:存放热更新下载的新资源,因为StreamingAssets改不了。

热更新加载逻辑:
    1. 启动时检查 PersistentDataPath 有没有新版本资源
    2. 没有 → 从 StreamingAssets 读取内置旧版本资源(首包自带)
    3. 有更新 → 从服务器下载新AB包存入 PersistentDataPath,之后优先从这里加载

### 2.3 AB 包加载机制

```
ABMgr (单例)
  │
  ├── LoadAB(abName)
  │   ├── 加载主包 → 读取 AssetBundleManifest (获取依赖关系)
  │   ├── 递归加载所有依赖包 (避免重复加载)
  │   └── 缓存到 Dictionary<string, AssetBundle>
  │
  ├── LoadRes<T>(abName, resName)
  │   从缓存中加载资源，GameObject 自动 Instantiate
  │
  └── LoadSceneFromAB(abName, sceneName)
      从 AB 包加载场景
```

## 三、Lua 热更新初始化

### 3.1 LuaMgr 初始化

```
LuaMgr.Init()
  │
  ├── new LuaEnv()                    → 创建 XLua 解析器
  ├── Global.Set("LoadSceneFromAB")   → 注册 AB 场景加载函数
  ├── AddLoader(MyCustomLoaderFormAB) → 注册自定义加载器
  │   (从 lua AB 包读取 .lua.txt 文件)
  ├── doString("require('GameLoop')") → 加载 Lua 帧循环调度器
  ├── 提取全局函数: Start / Update / Destroy
  └── 创建 LuaLoopDriver (MonoBehaviour)
       → Start()  → 调用 Lua Start
       → Update() → 调用 Lua Update + LuaEnv.Tick()
       → OnDestroy() → 调用 Lua Destroy
```

### 3.2 Lua 加载器

`MyCustomLoaderFormAB` 从 AB 包加载 Lua 脚本：
- 接收 `filepath`（如 `"Main"`）
- 提取文件名，补全为 `"Main.lua.txt"`
- 通过 `ABMgr.LoadRes<TextAsset>("lua", "Main.lua.txt")` 加载
- 返回 `byte[]` 给 XLua

### 3.3 Lua 入口 Main.lua

根据当前场景分流：

```
Main.lua
  │
  ├── 场景 == "Main"
  │   ├── require("UI/MainPanel")
  │   ├── require("UI/LevelSelect")
  │   ├── MainPanel:Init("panel_Main")
  │   └── 注册 Update 到 GameLoop
  │
  └── 场景 == 关卡 (Level1~4)
      ├── require("UI/GameOver")
      ├── require("UI/GamePause")
      ├── require("UI/GamePanel")
      ├── require("UI/SceneData")
      ├── require("PlayerCtrl"):new()    → 玩家状态机
      ├── require("Coin")                → 金币旋转
      ├── require("Fade")                → 场景切换渐入渐出
      └── 各模块注册 Update 到 GameLoop
```

## 四、热修复机制 (XLua Hotfix)

C# 类标记 `[Hotfix]` 特性后，其方法可在 Lua 侧被动态替换：

```lua
-- Lua 侧替换 C# 方法示例
xlua.hotfix(CS.PlayerMotor, "Jump", function(self)
    -- 新的跳跃逻辑，无需重新编译客户端
    self:_Jump()
end)
```

当前项目中标记了 `[Hotfix]` 的类：
- `PlayerMotor` — 角色运动控制

## 五、场景切换与重启

游戏在场景切换时需要完全重启 Lua 环境：

```
GameManager.OnSceneUnloaded()
  │
  └── LuaMgr.DoRestart(回调)
      │
      ├── 断开 Lua 函数引用 (Start/Update/Destroy = null)
      ├── 清理 Button.onClick 监听 (Lua 委托可能被持有)
      ├── 清理 DOTween 回调
      ├── 销毁 LuaLoopDriver 对象
      ├── 等待一帧 (确保 Unity Destroy 队列处理完毕)
      ├── 双轮 GC.Collect() + WaitForPendingFinalizers()
      ├── LuaEnv.Tick() + LuaEnv.Dispose()
      └── 重新 Init()
          → 回调中: SetToLua() + DoLuaFile("Main")
```
