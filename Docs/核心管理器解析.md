# 核心管理器解析：ABMgr 与 LuaMgr

---

## 一、ABMgr — AssetBundle 管理器

### 1.1 定位

`ABMgr` 继承自 `SingletonAutoMono<ABMgr>`，是一个全局唯一的 **MonoBehaviour 单例**，负责项目中所有 AB 包的加载、缓存、依赖解析、资源加载和卸载。

### 1.2 类图

```
SingletonAutoMono<ABMgr>  (泛型单例基类)
    │
    └── ABMgr
            ├── mainAB: AssetBundle          (主包)
            ├── manifest: AssetBundleManifest (依赖清单)
            └── abDic: Dictionary<string, AssetBundle>   (已加载包的缓存)
```

### 1.3 初始化机制

继承的 `SingletonAutoMono<T>` 基类在首次 `GetInstance()` 时：

1. `FindObjectOfType<T>()` 查找场景中是否已有实例
2. 若没有，创建新 GameObject 并 `AddComponent<T>()`
3. 设置 `DontDestroyOnLoad()` 保证过场景不被销毁

### 1.4 核心字段

| 字段 | 类型 | 作用 |
|------|------|------|
| `mainAB` | `AssetBundle` | 主包（即 AB 包清单文件），延迟加载，首次 `LoadAB()` 时初始化 |
| `manifest` | `AssetBundleManifest` | 从主包中加载的依赖清单，用于查询 AB 包间的依赖关系 |
| `abDic` | `Dictionary<string, AssetBundle>` | 已加载的 AB 包缓存，防止重复加载报错 |

### 1.5 AB 包加载机制（图解）

```
LoadAB("ui")
    │
    ├── mainAB == null?
    │   ├── AssetBundle.LoadFromFile(GetABPath("AB"))  → 加载主包
    │   └── mainAB.LoadAsset<AssetBundleManifest>(...) → 读取依赖清单
    │
    ├── manifest.GetAllDependencies("ui")              → 获取 ui 包的所有依赖
    │   └── 遍历依赖数组
    │       └── abDic 中没有？→ LoadFromFile 加载并加入缓存
    │
    └── abDic 中没有 "ui" 包？
        └── LoadFromFile 加载 "ui" 包并加入缓存
```

#### 路径策略：`GetABPath(abName)`

```
GetABPath(abName)
    │
    ├── persistentDataPath + "/" + abName  存在？
    │   └── 返回 persistent 路径 (下载更新的 AB 包)
    │
    └── 不存在 → 返回 streamingAssetsPath 路径 (打包内置的 AB 包)
```

#### 主包名固定为 "AB"

```csharp
private string MainABName => "AB";
```
注释中保留了 iOS/Android/PC 的平台分支设计，但当前统一为 `"AB"`，不做平台后缀区分。

### 1.6 资源加载方法

| 方法 | 同步/异步 | 说明 |
|------|-----------|------|
| `LoadRes(abName, resName)` | 同步 | 无类型，返回 `Object`，GameObject 自动 `Instantiate` |
| `LoadRes(abName, resName, type)` | 同步 | 指定 Type |
| `LoadRes<T>(abName, resName)` | 同步 | 泛型版本 |
| `LoadResAsync(abName, resName, callback)` | 异步 | 协程实现，`AssetBundleRequest` |
| `LoadResAsync<T>(...)` | 异步 | 泛型异步版本 |

**重要约定**：所有方法都约定如果资源是 `GameObject`，会在内部自动 `Instantiate`。这意味着调用方拿到的一定是场景中的实例，而不是原始预制体。

### 1.7 关键设计细节

```
LoadAB 中的依赖处理                  资源加载中的自动实例化
────────────────                    ──────────────────────
LoadAB("ui")                         Object obj = abDic[abName].LoadAsset(resName)
  ├── 先加载依赖包 (循环)             if (obj is GameObject)
  ├── for 循环中重复检查                  return Instantiate(obj);  ← 自动实例化
  │   if(!abDic.ContainsKey(...))     else
  └── 再加载目标包                         return obj;              ← 保持原样
      if(!abDic.ContainsKey(abName))
```

### 1.8 场景加载

```csharp
LoadSceneFromAB(abName, sceneName)
```
从 AB 包加载 Unity 场景，路径硬编码为 `"Assets/Scenes/" + sceneName + ".unity"`。场景必须已从 Build Settings 中移除并设置 AB Label。

### 1.9 卸载策略

| 方法 | 作用 |
|------|------|
| `UnLoad(abName)` | 卸载单个包，`Unload(false)` — 不卸载已实例化的资源对象 |
| `ClearAB()` | 全局卸载所有包 + 清空字典 + 释放主包引用 |

## 二、LuaMgr — Lua 引擎管理器

### 2.1 定位

`LuaMgr` 继承自 `BaseManager<LuaMgr>`（非 MonoBehaviour 单例），是整个项目中 **C# 与 Lua 交互的桥梁**。负责 Lua 引擎生命周期、脚本加载、帧循环调度、场景切换重启。

### 2.2 类图

```
BaseManager<LuaMgr>  (泛型单例基类)
    │
    └── LuaMgr
        ├── luaEnv: LuaEnv          (XLua 解析器)
        ├── luaStart: Action        (Lua 侧的 Start 函数)
        ├── luaUpdate: Action       (Lua 侧的 Update 函数)
        ├── luaDestroy: Action      (Lua 侧的 Destroy 函数)
        └── driver: LuaLoopDriver   (挂载在场景对象上的 MonoBehaviour 驱动)
                │
                └── LuaLoopDriver: MonoBehaviour
                    ├── Start()   → LuaMgr.Start()
                    ├── Update()  → LuaMgr.Update()
                    └── OnDestroy() → LuaMgr.Destroy()
```

### 2.3 初始化流程

```
Init()
    │
    ├── new LuaEnv()                              ── 创建 XLua 虚拟机
    │
    ├── Global.Set("LoadSceneFromAB", 回调)        ── 注册 C# 函数到 Lua
    │   (让 Lua 能调用 ABMgr.LoadSceneFromAB)
    │
    ├── luaEnv.AddLoader(MyCustomLoaderFormAB)     ── 注册自定义加载器
    │   (取代默认的 Resources 加载，改为从 AB 包读 .lua.txt)
    │
    ├── luaEnv.DoString("require('GameLoop')")     ── 加载 Lua 帧循环调度器
    │
    ├── 提取 Lua 全局函数:
    │   ├── luaStart   = Global.Get<Action>("Start")
    │   ├── luaUpdate  = Global.Get<Action>("Update")
    │   └── luaDestroy = Global.Get<Action>("Destory")    ← 注意: 拼写为 Destory
    │
    └── 创建 LuaLoopDriver
        ├── new GameObject("LuaMgrDriver")
        ├── DontDestroyOnLoad()
        └── driver = go.AddComponent<LuaLoopDriver>()
```

### 2.4 Lua 加载器机制

#### MyCustomLoaderFormAB

```
MyCustomLoaderFormAB(ref filepath)
    │
    ├── 输入: filepath = "Main"
    ├── Path.GetFileName → "Main"
    ├── ABMgr.LoadRes<TextAsset>("lua", "Main.lua.txt")
    │       │
    │       └── 从 lua AB 包中加载 Main.lua.txt
    │
    ├── 成功 → 返回 TextAsset.bytes (byte[])
    │
    └── 失败 → 调试日志 + 打印 AB 包资源列表
```

**为什么存为 .lua.txt？**
- Unity 的 TextAsset 默认识别 `.txt` 后缀
- 加 `.lua` 前缀保留语义，加 `.txt` 后缀让 Unity 当文本资源处理

#### 注释掉的 MyCustomLoader

```csharp
private byte[] MyCustomLoader(ref string filepath)
```
从 `Assets/Lua/` 文件系统直接加载，用于纯编辑器调试（不走 AB 包）。当前被注释，启用的是 AB 包加载器。

### 2.5 帧循环驱动

```
Unity 主循环                     LuaMgr                     Lua 侧
──────────                     ──────                     ──────
                              GameLoop (Lua 模块)
LuaLoopDriver.Update()                                  Module1:Update()
    │                               │                        Module2:Update()
    └── LuaMgr.Update()           调度                       ...
        ├── luaUpdate?.Invoke()  ─→  GameLoop.Update()  ─→  ...
        └── luaEnv?.Tick()       ─→  增量 GC                       │
                                                          ModuleN:Update()
```

`GameLoop.lua` 是自定义的帧更新调度器，各模块通过 `RegisterLoop(start, update, destroy)` 注册自己的生命周期函数。

### 2.6 Lua 环境重启（场景切换）

场景切换时 Lua 环境需要完全重建，这是因为 XLua 的 `LuaEnv` 一旦创建就和场景状态绑定，切场景不重置的话，旧的 Lua 对象引用会悬空或内存泄漏。

```
DoRestart(onCompleted)
    │
    ├── ① 断开 Lua 函数引用
    │   luaStart = null
    │   luaUpdate = null
    │   luaDestroy = null
    │
    ├── ② 清理按钮监听
    │   Button.onClick.RemoveAllListeners()
    │   (Lua 注册的委托可能被 C# 持有，不清理会造成死引用)
    │
    ├── ③ 清理 DOTween 回调
    │   DOTween.KillAll()
    │
    ├── ④ 销毁 LuaLoopDriver
    │   GameObject.Destroy(driver.gameObject)
    │
    ├── ⑤ yield return null (等一帧，让 Destroy 队列处理完)
    │
    ├── ⑥ 双轮 GC
    │   GC.Collect() → WaitForFinalizers() → GC.Collect() → WaitForFinalizers()
    │   (首次收集可达垃圾，第二次收集 Finalizer 产生的新垃圾)
    │
    ├── ⑦ 销毁旧 LuaEnv
    │   luaEnv.Tick() → luaEnv.Dispose() → luaEnv = null
    │
    ├── ⑧ 重新 Init()
    │   全新 LuaEnv + Loader + GameLoop + Driver
    │
    └── ⑨ onCompleted?.Invoke()
        (GameManager 在此回调中 SetToLua + DoLuaFile("Main"))
```

### 2.7 公开 API

| API | 作用 |
|-----|------|
| `Init()` | 初始化 Lua 引擎 |
| `Start()` | 触发 Lua Start |
| `Update()` | 触发 Lua Update + LuaEnv.Tick |
| `Destroy()` | 触发 Lua Destroy |
| `Global` (属性) | 返回 `LuaTable`，C# 获取 Lua 全局变量/函数 |
| `DoLuaFile(fileName)` | 执行 `require(fileName)` |
| `DoString(luaScript)` | 直接执行 Lua 代码字符串 |
| `Tick()` | 手动触发 LuaEnv 增量 GC |
| `Dispose()` | 完整销毁 Lua 环境 |
| `DoRestart(onCompleted)` | 协程式重建 Lua 环境（场景切换用） |

## 三、两个管理器如何协作

### 3.1 Lua 从 AB 包加载

```
LuaMgr.Init()
    │
    └── AddLoader(MyCustomLoaderFormAB)
            │
            └── require("Main")
                    │
                    └── ABMgr.LoadRes<TextAsset>("lua", "Main.lua.txt")
                            │
                            ├── ABMgr.LoadAB("lua")   → 加载 lua AB 包
                            └── abDic["lua"].LoadAsset<TextAsset>("Main.lua.txt")
```

### 3.2 Lua 通过 C# 加载场景

```
Lua 侧调用 LoadSceneFromAB(abName, sceneName)
    │
    └── C# 侧注册的委托
            │
            └── ABMgr.GetInstance().LoadSceneFromAB(abName, sceneName)
                    │
                    ├── LoadAB(abName)
                    └── SceneManager.LoadScene("Assets/Scenes/" + sceneName + ".unity")
```

### 3.3 触发器事件

```
PlayerMotor.OnTriggerEnter()
    │
    ├── LuaMgr.Global.Get<LuaFunction>("OnPlayerTrigger")
    └── fn.Call(other.gameObject)
```

将碰撞事件转发给 Lua 端决策（具体逻辑如扣血、加分等由 Lua 实现）。

## 四、整体数据流

```
    开发期                         运行时
    ────                          ────
  .lua 文件          ──打包──►  lua AB 包 (.lua.txt)
                                    │
                                    ├── ABMgr.LoadAB("lua")
                                    │       │
  C# 方法 (场景加载等) ──注册──►  LuaMgr.Global.Set(...) ──► Lua 侧调用
                                    │
  预制体/模型         ──打包──►  game AB 包 ──► ABMgr.LoadRes<GameObject>(...)
                                    │
  UI 资源            ──打包──►  ui AB 包   ──► ABMgr.LoadRes<GameObject>(...)
                                    │
  场景数据           ──打包──►  scene AB 包 ──► ABMgr.LoadSceneFromAB(...)
```
